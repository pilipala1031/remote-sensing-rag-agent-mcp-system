# 遥感语义分割典型难点与解决思路

本文件总结遥感语义分割相比自然图像分割的核心难点，以及对应的工程与算法对策，用于知识库问答。

## 一、尺度变化剧烈（Scale Variation）

### 难点

- 同一类目标在影像中尺寸跨度极大：iSAID 中船只可能 20 像素，也可能 2000 像素。
- 固定输入分辨率（如 512×512）训练的模型对极端尺度泛化差。
- 遥感影像单张尺寸极大（DeepGlobe 2448×2448，Potsdam 6000×6000），必须裁剪训练，裁剪可能切断大目标。

### 对策

1. **多尺度训练 / 推理**：训练时随机 resize 到不同尺度；推理时做多尺度 TTA（Test Time Augmentation）融合。
2. **空洞空间金字塔池化（ASPP）**：DeepLabV3+ 的核心模块，用不同 dilation rate 并行聚合多尺度上下文。
3. **金字塔池化模块（PPM）**：PSPNet 的设计，1×1 / 2×2 / 3×3 / 6×6 四个分支。
4. **HRNet backbone**：始终保持高分辨率分支，多分辨率并行融合，对尺度敏感任务效果好。
5. **滑窗推理 + 重叠融合**：裁剪推理时保留 20%~50% overlap，重叠区域取平均或加权，减少边界伪影。

## 二、类别极度不平衡（Class Imbalance）

### 难点

- 遥感影像中"背景 / 裸地 / 林地"可能占 80% 以上像素，"车辆 / 桥梁 / 船只"占比 <1%。
- 直接训练会让模型偏向大类，小类 IoU 接近 0。
- LoveDA 中 background 类的 IoU 通常远高于 agriculture / forest。

### 对策

1. **加权交叉熵（Weighted CE）**：按类别频率倒数赋权，PyTorch 中 `CrossEntropyLoss(weight=...)`。
2. **Focal Loss**：Lin et al. 2017，对易分样本降权 `(1 - p_t)^γ`，聚焦难分小类，γ 常取 2。
3. **Dice Loss / Tversky Loss**：直接优化 IoU 类指标，对小类更敏感；Dice Loss = 1 - 2|X∩Y| / (|X|+|Y|)。
4. **混合损失**：`loss = α × CE + β × Dice` 是分割任务常用组合，兼顾像素级监督与区域级优化。
5. **过采样 / 类平衡采样**：训练时对小类密集的影像块（tile）加权采样。
6. **OHEM（Online Hard Example Mining）**：只对损失最高的像素反传梯度。

## 三、边界模糊（Boundary Ambiguity）

### 难点

- 建筑与道路、林地与农业用地的过渡区域往往光谱相近，边界像素分类错误率高。
- 标准 mIoU 对边界像素不敏感，但视觉上边界错乱严重影响制图质量。

### 对策

1. **Boundary Loss**：Kervadec et al. 2019，对边界像素额外惩罚，常用于医学/遥感分割。
2. **CRF 后处理**：条件随机场（DenseCRF）利用影像颜色与位置平滑预测，DeepLab v1/v2 使用。
3. **边界感知特征**：额外加一个边界检测分支，显式监督地物边缘。
4. **高分辨率特征保留**：U-Net skip connection、DeepLabV3+ 的 low-level featuere 融合都是为此设计。

## 四、跨域泛化差（Domain Gap）

### 难点

- 模型在 LoveDA Urban 训练，到 Rural 推理 mIoU 通常掉 10~20 个点。
- 不同传感器（WorldView-3 vs Google Earth vs 无人机）光谱与辐射特性差异大。
- 季节、光照、地域变化都会引入域偏移。

### 对策

1. **域随机化（Domain Randomization）**：训练时做强增广（颜色抖动、随机擦除、CutMix），提升鲁棒性。
2. **域适应（Domain Adaptation）**：DANN、CycleGAN 风格迁移、对抗式特征对齐。
3. **自训练 / 伪标签**：在目标域上用模型预测高置信度像素作伪标签迭代训练。
4. **多源数据混合训练**：直接混合 LoveDA + DeepGlobe + Potsdam 一起训练，是最简单有效的泛化手段。

## 五、旋转不变性（Rotation Invariance）

### 难点

- 航空/卫星影像中目标方向任意：船只朝向、车辆停放、农田走向都不固定。
- CNN 的卷积核本质上对方向敏感，水平/垂直结构偏好明显。
- 自然图像预训练模型对"任意旋转"鲁棒性差。

### 对策

1. **旋转增广**：训练时随机旋转 0/90/180/270° 或任意角度。
2. **旋转等变网络**：OctConv、Group Convolution 显式建模旋转等变性。
3. **旋转一致性自监督**：让模型对同一影像的不同旋转版本输出一致特征。

## 六、标注稀缺与噪声（Label Scarcity & Noise）

### 难点

- 遥感像素级标注成本极高，iSAID 标注耗费数千人工小时。
- 半监督、弱监督是遥感分割的常态。
- 标注本身有噪声（人工勾绘边界不准、类别混淆）。

### 对策

1. **半监督学习**：Mean Teacher、FixMatch，用一致性正则利用无标注影像。
2. **自监督预训练**：SatMAE、ScaleMAE 在大量无标注遥感影像上 MAE 预训练，再下游微调，比 ImageNet 预训练涨点明显。
3. **弱监督**：图像级标签 + CAM（Class Activation Map）初始化，再迭代优化。
4. **主动学习**：选模型最不确定的 tile 优先标注，降低标注预算。

## 七、大尺度推理的工程挑战

### 难点

- Potsdam 单张 6000×6000，显存无法直接放下，必须滑窗推理。
- 推理速度、显存、精度三者权衡。

### 对策

1. **滑窗推理（Sliding Window）**：固定 patch 大小（如 512×512）+ stride 重叠，重叠区取平均。
2. **模型量化**：FP16 / INT8 推理，TensorRT 加速，显存减半、速度翻倍。
3. **模型剪枝 / 蒸馏**：用大模型（Swin-L）蒸馏小模型（SegFormer-B0）做部署。
4. **多 GPU / 批处理推理**：把多个 tile 组 batch 并行推理。

## 八、光谱与多模态利用

### 难点

- 多光谱影像有 NIR（近红外）、SWIR（短波红外）等额外通道，单靠 RGB 浪费信息。
- NIR 对植被分割极有帮助（NDVI 直接由 NIR 与 Red 计算）。
- DSM/DEM 高程对建筑、树的区分关键。

### 对策

1. **多通道输入**：修改第一层卷积输入通道数（如 4 通道 RGBN），用零填充或重复权重初始化。
2. **多模态融合**：RGB 分支 + DSM 分支双流网络，中间层特征拼接。
3. **光谱指数通道**：预计算 NDVI、NDWI、NDBI 作为额外输入通道，对地表覆盖任务涨点稳定。

## 九、总结：遥感分割调优 checklist

按优先级排序的工程技巧（多数任务适用）：

1. ✅ 强增广（翻转 + 旋转 + 颜色抖动 + CutMix）—— 提升泛化，几乎零成本
2. ✅ Dice + CE 混合损失 —— 缓解类别不平衡
3. ✅ 滑窗推理 + overlap 融合 —— 大图必备
4. ✅ 多尺度 TTA —— 稳定涨 1~3 mIoU
5. ✅ 遥感预训练 backbone（如有）—— 比 ImageNet 涨 2~5 mIoU
6. ✅ per-class IoU 分析 —— 定位短板类别，针对性增广/采样
7. ✅ 类别加权 / Focal Loss —— 小类 IoU 低时必试
8. ⬜ CRF 后处理 —— 边界要求高的制图任务
9. ⬜ 模型集成 —— 竞赛刷分，工程部署慎用
